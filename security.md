# Tietoturva

Olemme tehneet parhaamme valitaksemme esimerkkeihin ja tehtäviin vain välttämättömät ja yleisesti luotettavina pidetyt riippuvuudet. Tästä huolimatta, kuten kaikessa ohjelmistokehityksessä, aina on mahdollisuus virheisiin tai tietoturva-aukkoihin.

Myös tunnetuista ja laajasti käytetyistä kirjastoista paljastuu ajoittain haavoittuvuuksia. Tietoturva ei ole kertaluonteinen tehtävä, vaan jatkuva prosessi. Haavoittuvuuksien seuraaminen, niihin reagoiminen sekä riippuvuuksien päivittäminen ovat tärkeä osa vastuullista projektikehitystä. Nämä käytännöt ovat suositeltavia käytäntöjä kaikissa ohjelmistoprojekteissa.

Jos sinulla on kysyttävää, huomaat mahdollisen tietoturvaongelman oppimateriaaleissa tai epäilyttävää toimintaa projektin tehtävissä tai niiden riippuvuuksissa, ota viipymättä yhteyttä opettajaasi ja projektin ylläpitäjään.

Kiitos, että autat pitämään projektin turvallisena!


# Security

We have made every effort to select only necessary and generally trusted dependencies for our projects. Nevertheless, as with all software development, there is always the possibility of errors or security vulnerabilities.

Even well-known and widely used libraries may be found to contain vulnerabilities. Security is not a one-time decision, but an ongoing process. Monitoring for vulnerabilities, responding to them, and keeping dependencies up to date are essential parts of responsible project development. These practices are advisable in all software projects.

If you have any questions or notice a potential security issue or suspicious behavior in the project tasks or their dependencies, please contact your instructor and the project maintainer without delay.

Thank you for helping keep the project secure!
